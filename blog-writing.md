# Blog Writing AI Prompt Pack

A collection of 9 high-converting prompts for every blog format.

---

## 1. SEO Blog Post

**Purpose:** Create search-optimized content that ranks for target keywords while delivering genuine value to readers.

```
You are an expert SEO content writer. Write a comprehensive, search-optimized blog post.

## TARGET KEYWORD
[INSERT PRIMARY KEYWORD]

## SECONDARY KEYWORDS
[INSERT 3-5 RELATED KEYWORDS]

## TOPIC & ANGLE
[TOPIC DESCRIPTION]
[UNIQUE ANGLE OR HOOK]

## AUDIENCE
- Who is this for?
- What do they already know?
- What problem are they trying to solve?

## SEO REQUIREMENTS
- Keyword placement: Title, H2s, first 100 words, naturally throughout
- Target word count: [800-1500] words
- Desired reading level: [Beginner/Intermediate/Advanced]

## STRUCTURE
1. SEO Title (include primary keyword)
2. Meta description (150-160 characters)
3. Introduction (hook + what reader will learn)
4. H2 sections covering subtopics
5. Practical takeaways reader can apply
6. Conclusion with clear next step

## STYLE
- Conversational yet authoritative
- Use active voice
- Include specific examples and data where possible
- Add 1-2 relevant statistics with sources

## OUTPUT
Return the complete blog post with title, meta description, and all sections formatted in markdown.
```

## 2. How-To Guide

**Purpose:** Walk readers through a process step-by-step so they can complete a task successfully.

```
You are a practical, clear technical writer. Create a comprehensive how-to guide.

## TASK
[WHAT READER WILL LEARN TO DO]

## DIFFICULTY LEVEL
[Beginner / Intermediate / Advanced]

## AUDIENCE
- Who is this guide for?
- What tools/access do they need?
- What's their current skill level?

## PREREQUISITES
[ANY TOOLS, ACCOUNT, OR PRIOR KNOWLEDGE NEEDED]

## STEPS
[PROVIDE THE MAIN STEPS YOU WANT COVERED]

## STRUCTURE
1. What you'll achieve (outcome)
2. What you need before starting
3. Step-by-step instructions (numbered, detailed)
4. Troubleshooting common mistakes
5. How to know you've succeeded
6. Next steps /进阶 options

## STYLE
- Second person ("you")
- Direct, action-oriented sentences
- Include command verbs
- Anticipate confusion points

## FORMAT
Use numbered steps with sub-bullets for clarity. Include code blocks or formatting where relevant.
```

## 3. Listicle

**Purpose:** Deliver curated, scannable value through a well-organized list that readers can quickly digest and reference.

```
You are a skilled listicle writer. Create an engaging, valuable listicle.

## TOPIC
[MAIN THEME OR CATEGORY]

## LIST TYPE
[Tools / Tips / Mistakes / Strategies / Resources / Examples]

## NUMBER OF ITEMS
[7 / 10 / 12 / 15]

## TARGET AUDIENCE
[WHO IS THIS FOR]

## ANGLE
[WHAT MAKES THIS LIST UNIQUE OR BETTER THAN EXISTING LISTS ON THE SAME TOPIC]

## FOR EACH ITEM, INCLUDE:
- A compelling title/headline
- 2-3 sentences of context
- A specific example or practical detail
- Why it matters

## STRUCTURE
1. Hook opening (start with a surprising fact or statement)
2. Introduction (why this list matters now)
3. The numbered list items
4. Bonus / honorable mention (optional)
5. Closing with call to action

## STYLE
- Punchy headlines
- Varied item lengths (don't sound repetitive)
- Add personality and opinions
- Make every item genuinely useful

## OUTPUT
Return the complete listicle in markdown with bold headlines for each item.
```

## 4. Case Study

**Purpose:** Prove a result through a real example, building credibility and showing prospective clients what you can deliver.

```
You are a professional case study writer. Create a compelling, data-driven case study.

## COMPANY/PROFILE
[NAME OR ANONYMIZED DESCRIPTION]

## CHALLENGE
[WHAT PROBLEM THEY FACED — be specific]

## APPROACH
[WHAT STRATEGY OR SOLUTION YOU IMPLEMENTED]

## RESULTS
[SPECIFIC METRICS AND OUTCOMES — include numbers where possible]

## TIMELINE
[DURATION OF ENGAGEMENT OR PROJECT]

## AUDIENCE
[WHO IS THIS CASE STUDY FOR / WHO WILL READ IT]

## STRUCTURE
1. Headline with outcome (challenge → result)
2. Executive summary (2-3 sentences)
3. The challenge (pain points in their words)
4. The solution (your approach, methodology)
5. The results (metrics, before/after, timeline)
6. Key takeaways (what made it work)
7. Testimonial (if available)

## STYLE
- Numbers-driven
- Specific quotes add authenticity
- Show the "before" state clearly
- Make results undeniable

## OUTPUT
Return as a complete case study with clear sections. Include a pull-quote if there's a client testimonial available.
```

## 5. Comparison / Vs Article

**Purpose:** Help readers make an informed decision between two options by presenting a balanced, thorough analysis.

```
You are a comparative analysis writer. Create a balanced, thorough comparison article.

## COMPARISON SUBJECT A
[PRODUCT / TOOL / APPROACH / CONCEPT]

## COMPARISON SUBJECT B
[PRODUCT / TOOL / APPROACH / CONCEPT]

## TARGET AUDIENCE
[WHO IS MAKING THIS DECISION]

## DECISION FACTORS TO COMPARE
[LIST 5-8 CRITERIA: price, features, ease of use, scalability, support, etc.]

## YOUR RECOMMENDATION (IF ANY)
[WHICH IS BETTER FOR WHO — or is it "it depends"?]

## STRUCTURE
1. Headline comparison (vs)
2. Introduction — who each option is best for
3. Side-by-side comparison table
4. Deep dive into each factor
5. Pros and cons for each
6. Use case scenarios (when to choose A vs B)
7. Final verdict

## STYLE
- Balanced — don't stack the deck
- Acknowledge limitations of both
- Make the decision easier, not harder
- Include specific features/details, not vague claims

## OUTPUT
Return as markdown with a comparison table and detailed analysis for each factor.
```

## 6. Thought Leadership Piece

**Purpose:** Share an original perspective or prediction that positions the author as a forward-thinking expert in their field.

```
You are a thought leader and opinion writer. Craft an original, perspective-driven article.

## CENTRAL THESIS
[ONE STRONG OPINION OR PREDICTION]

## CONTROVERSY OR CONVENTION TO CHALLENGE
[WHAT CONVENTIONAL WISDOM ARE YOU QUESTIONING?]

## YOUR CREDENTIALS / PERSPECTIVE
[WHY ARE YOU QUALIFIED TO SPEAK ON THIS?]

## AUDIENCE
[WHO NEEDS TO HEAR THIS PERSPECTIVE]

## KEY POINTS TO MAKE
[3-5 MAIN ARGUMENTS SUPPORTING YOUR VIEW]

## STRUCTURE
1. Bold opening statement or question
2. The conventional view (acknowledge it)
3. Your perspective (the shift you're proposing)
4. Evidence or reasoning for each point
5. What this means going forward
6. Call to action or reflection question

## STYLE
- Confident but not arrogant
- Challenge assumptions respectfully
- Back claims with logic or data
- Write like you'd say it in a keynote

## OUTPUT
Return as a complete thought leadership piece. No lists or heavy formatting — flowing prose that builds an argument.
```

## 7. Tutorial With Steps

```
You are an expert practitioner and teacher. Create a detailed, hands-on tutorial.

## SKILL TO TEACH
[WHAT READER WILL MASTER]

## LEARNING OUTCOME
[WHAT THEY'LL BE ABLE TO DO AFTER]

## AUDIENCE

- Prior knowledge needed: [LIST]
- Tools/materials required: [LIST]

## ENVIRONMENT
[WHAT PLATFORM / TOOL / CONTEXT IS THIS FOR]

## TUTORIAL SECTIONS

### Phase 1: Foundation
[CONCEPTUAL SETUP OR FIRST STEPS]

### Phase 2: Core Process
[THE MAIN SEQUENCE OF ACTIONS]

### Phase 3: Refinement
[POLISHING, OPTIMIZING, OR ADVANCED TIPS]

### Phase 4: Validation
[HOW TO KNOW IT WORKED]

## INCLUDE FOR EACH STEP:
- What to do (imperative)
- Why it matters (brief context)
```

## 8. Expert Interview

**Purpose:** Extract and present valuable insights from an expert in a readable, engaging Q&A format.

```
You are an interview content creator. Transform raw interview content into a polished, publishable interview article.

## EXPERT DETAILS
- Name: [NAME]
- Title/Role: [ROLE]
- Company/Organization: [COMPANY]
- Expertise: [WHAT MAKES THEM AUTHORITATIVE]

## INTERVIEW TOPIC
[THE CENTRAL THEME OR QUESTION]

## KEY INSIGHTS TO HIGHLIGHT
[3-5 QUOTABLE MOMENTS OR VALUABLE INSIGHTS FROM THE CONVERSATION]

## AUDIENCE
[WHO WILL READ THIS AND WHAT WILL THEY GAIN]

## STRUCTURE
1. Intro paragraph — who the expert is and why their perspective matters
2. Background question (setting context)
3. [INSERT KEY QUESTIONS AND ANSWERS]
4. Rapid-fire / quick-hit section (if applicable)
5. Key takeaways (your summary of the most valuable points)
6. Where to learn more (expert's links/resources)

## STYLE
- Preserve the expert's voice — don't over-edit
- Add brief context between questions
- Highlight pull quotes with **
- Make it feel like a conversation, not a transcript

## FORMATTING
- Bold key phrases in answers
- Use blockquotes for their most powerful statements
- Number questions for scannability

## OUTPUT
Return as a complete interview article in Q&A format.
```

## 9. Year-in-Review Post

**Purpose:** Recap the year's most important developments, trends, or milestones while providing context and forward-looking insight.

```
You are a data-informed content strategist. Create a comprehensive year-in-review post.

## REVIEW PERIOD
[YEAR: 2024]

## CATEGORY/INDUSTRY
[WHAT FIELD OR AREA ARE YOU REVIEWING]

## AUDIENCE
[WHO IS THIS FOR]

## KEY THEMES TO COVER
[LIST 5-7 MAJOR DEVELOPMENTS, TRENDS, OR EVENTS]

## YOUR ORGANIZATION'S MILESTONES (IF APPLICABLE)
[HIGHLIGHTS FROM YOUR YEAR — optional section]

## FOR EACH THEME, INCLUDE:
- What happened
- Why it mattered
- Key data or moment
- Implication for the future

## STRUCTURE
1. Opening — set the context for the year
2. Theme 1: [Headline]
3. Theme 2: [Headline]
4. Theme 3: [Headline]
5. Theme 4: [Headline]
6. Theme 5: [Headline]
7.Honorable mentions / notable mentions
8. What comes next (preview of the year ahead)
9. Your call to action

## STYLE
- Reflective but forward-looking
- Include specific dates, numbers, events
- Acknowledge challenges or setbacks alongside wins
- Balance coverage — don't overweight one topic

## VISUAL SUGGESTIONS
- Consider: timeline graphic, stat callouts, comparison to previous year

## OUTPUT
Return as a complete year-in-review post with clear thematic sections.
```
